# Claude 工作量优化实施计划

## 目标

把 hybrid 多模型链路从“DeepSeek 聚类 + Claude 重新研究原始素材”调整为“DeepSeek 生成 editorialPacket + Claude 只做最终定稿”，降低 Claude 输入规模，同时保持 `output.js` 兼容的最终 JSON。

## 实施步骤

1. 补充单元测试：
   - `src/summarize/preprocess.test.js` 验证 `editorialPacket` 校验、默认值、长度统计、入选/丢弃统计。
   - `src/summarize/write-report.test.js` 验证 `inputMode: editorialPacket` 时 prompt 不包含大批原始素材，并记录输入规模。
   - 复用 `src/llm.test.js` 中 b.ai Claude 不传采样参数的测试。

2. 修改 `src/summarize/preprocess.js`：
   - 支持 `preprocess.outputMode: editorialPacket`。
   - 新增 `buildEditorialPacketUserPrompt` 和 `validateEditorialPacketResult`。
   - 输出并审计 `packetCharCount`、`selectedItemCount`、`droppedItemCount`。

3. 修改 `src/summarize/write-report.js`：
   - 支持 `writer.inputMode: editorialPacket`。
   - 默认 `includeRawSourceItems: false` 时不附加原始素材。
   - Claude prompt 改成“定稿”，明确不新增事实、不重新扩展素材范围。
   - 审计 `inputMode`、`includeRawSourceItems`、`inputCharCount`、`targetOutputChars`、`maxInputOutputRatio`。

4. 更新配置与文档：
   - `config/topics/global-geopolitical-conflicts-rss.yaml` 切到 `editorialPacket` 流程。
   - `config/topics/test-hybrid.yaml` 同步小样本流程，便于验证。
   - `CLAUDE.md` 增加多模型职责边界说明。

5. 验证：
   - 先运行新增相关测试。
   - 再运行 `npm test`。
   - 如环境变量齐全，运行 `npm run brief:test-hybrid` 和正式主题；如命令或环境缺失，在完成文档记录阻塞原因。

## 风险控制

- 对模型输出缺字段做默认值归一，不让后续 writer 崩溃。
- 保留旧 `clusters` 模式兼容未切换主题。
- `includeRawSourceItems` 只有显式为 `true` 才附加原始素材。
- packet 超限先审计并在 prompt 强约束，避免本地硬截断破坏 JSON 证据结构。
