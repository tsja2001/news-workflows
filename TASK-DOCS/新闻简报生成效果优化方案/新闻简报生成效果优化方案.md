# 新闻简报生成效果优化方案

## Context（为什么改）

RSS 抓取链路（`global-geopolitical-conflicts-rss.yaml`）目前抓取 60 条、生成正常，但实际产出（`2026-05-18-153021-全球地缘冲突速报（RSS版）.md`）有几个明显的体验问题：

1. **每条新闻一坨 150-200 字段落**：`keyDevelopments[i].detail` 是连续段落，没要点没小标题，扫读累。
2. **掺入无关条目**：60 条来源里夹着"印度学生车祸"、"NHS 护士调查"、"挪威取消售马导弹"、"朝鲜女足 U17 夺冠"、"Stacey Abrams 播客"等与地缘冲突弱相关甚至无关的内容。关键词过滤太宽，LLM 也没主动剔除。
3. **编辑评价不显眼**：实际已有大量评价（主编复盘整段 + detail 里夹带），但 ①集中在末尾 ②没"每条新闻独立点评" ③语气过克制（财新调调，temperature=0.3）。
4. **三段宏观叙述重叠**：`overview` / `context` / `editorReview` 把同一组判断说了三遍。
5. **prompt 硬编码**：风格、读者偏好、排除话题完全写死在 [src/summarize.js:39](src/summarize.js#L39)，yaml 没法配。

用户偏好（已确认）：
- 评价风格：**私人内参**（口语、立场鲜明、敢下判断，"说白了就是…")
- 关键变化每条拆三段「**发生了什么 / 为什么重要 / 编辑怎么看**」
- 开头加 **30 秒速读 TL;DR**
- 低关注度合并成**短讯**一行带过
- 不按地区分组（用户主动没选）
- 筛选**双层**：fetch 阶段 `excludeKeywords` 硬过滤 + LLM 阶段按读者画像主动剔除/降级
- **配置化**：在 yaml 加 `editorial` 段，让所有主题独立调风格

## 二、fetch 阶段：`excludeKeywords` 硬过滤

### [src/fetch/common.js](src/fetch/common.js)

在 `filterByKeywords`（第 27 行）之后新增姊妹函数：

```js
/**
 * 排除关键词过滤：标题或摘要命中任一关键词则丢弃（大小写不敏感）
 * @param {import('./types.js').NewsItem[]} items
 * @param {string[]} excludeKeywords
 * @returns {import('./types.js').NewsItem[]}
 */
export function filterByExcludeKeywords(items, excludeKeywords) {
  if (!excludeKeywords || excludeKeywords.length === 0) return items
  const lowered = excludeKeywords.map(k => k.toLowerCase())
  return items.filter(item => {
    const text = `${item.title} ${item.summary}`.toLowerCase()
    return !lowered.some(k => text.includes(k))
  })
}
```

在 `applyFilters`（第 73 行）的"关键词过滤"段（第 89-98 行）之后，**插入** exclude 阶段：

```js
// 排除关键词过滤

const droppedExclude = beforeExclude - result.length

}
if (auditor) {
  auditor.event('pipeline_filter', { stage: 'exclude_keyword', before: beforeExclude, after: result.length, dropped: droppedExclude })
}
```

### 新 `SYSTEM_PROMPT`（[src/summarize.js:39](src/summarize.js#L39) 整段替换）

```js
const DEFAULT_PERSONA = '资深国际新闻主编'
const DEFAULT_TONE = '专业克制，有判断但不情绪化'

function buildSystemPrompt(editorial = {}) {
  const persona = editorial.persona || DEFAULT_PERSONA
  const tone = editorial.tone || DEFAULT_TONE

  return `你是${persona}。你的工作不是替主流媒体复述新闻，而是替这位读者把事情吃透、说清。

要做的几件事：
1. **看清主线**：从一堆零散素材里抽出几条真正重要的变化，按"对读者重要"排序，不按"哪条最近"排序。
2. **替他过滤**：读者明确告诉过你他不关心哪些话题。即使素材里出现，也不要塞进 keyDevelopments；可以放到 briefs 一行带过，或者直接不提。
3. **替他判断**：每条变化下面除了"发生了什么"、"为什么重要"，再加一句"编辑怎么看"——你的立场、判断、提醒。可以口语，可以辛辣，但要有素材依据，不能凭空发挥。
4. **不装客观**：这份简报是给一个人看的，不是发稿。少用"或将"、"可能"、"有观察人士认为"这种官话；多用"说白了"、"我看…"、"别看…其实…"这种直接判断。

写作风格：${tone}

硬性规则：
- 只基于提供的素材，绝不编造事实或细节
- 中文输出
- 严格按 JSON schema 返回，不要带任何额外解释或 markdown 围栏（不要用 \`\`\`json）
- 字段没内容就返回空数组/空字符串，不要凑数`
}
```

### 新 `buildUserPrompt`（[src/summarize.js:59](src/summarize.js#L59) 整段替换）

```js
function buildUserPrompt(items, config) {
  const editorial = config.editorial || {}
  const interests = editorial.interests || []
  const excludeTopics = editorial.excludeTopics || []
  const tldrEnabled = editorial.tldr?.enabled !== false   // 默认开
  const tldrMax = editorial.tldr?.maxItems ?? 5
  const kdHigh = editorial.keyDevelopmentsLimit?.high ?? 5
  const kdMedium = editorial.keyDevelopmentsLimit?.medium ?? 3
  const lowHandling = editorial.lowAttentionHandling || 'brief'
  const mergeContext = editorial.mergeContextIntoOverview !== false  // 默认开

  const interestsBlock = interests.length
    ? `\n读者最关心的方向（优先呈现这些）：\n${interests.map(i => `- ${i}`).join('\n')}\n`
    : ''
  const excludeBlock = excludeTopics.length
    ? `\n读者明确不感兴趣的话题（请剔除或降到 briefs）：\n${excludeTopics.map(t => `- ${t}`).join('\n')}\n`
    : ''

  const itemsText = items.map((item, i) =>
    `[${i + 1}] 来源: ${item.source}
时间: ${item.publishedAt}
标题: ${item.title}
摘要: ${item.summary}
URL: ${item.url}`
  ).join('\n\n')

  // 注意：schema 中的注释会被 JSON.parse 报错，这里只是给 LLM 看的"伪 JSON"
  return `请基于下面 ${items.length} 条关于"${config.title}"的新闻，撰写一份编辑简报。
${interestsBlock}${excludeBlock}
返回 JSON 格式（严格遵守，不要加 markdown 围栏）：
{
  ${tldrEnabled ? `"tldr": ["30 秒速读 bullet，3-${tldrMax} 条，每条 30-50 字，覆盖本期最该知道的事"],` : ''}
  "overview": "本期概览，1 段 100-150 字。${mergeContext ? '同时把宏观背景（更大格局、共同主线）融进来，不再单独出 context。' : ''}",
  "keyDevelopments": [
    {
      "title": "短句标题，不超过 25 字",
      "what": "【发生了什么】1-2 句，60-100 字。只讲事实，不评价",
      "why": "【为什么重要】1-2 句，60-100 字。讲影响、利益相关、结构性意义",
      "editorTake": "【编辑怎么看】1-2 句，60-120 字。带立场的判断、提醒、吐槽，可口语化",
      "importance": "high 或 medium"
    }
  ],
  ${mergeContext ? '' : '"context": "整体背景分析 100-200 字，'}
  "briefs": ["低关注度短讯。格式：【主题/地区】 一句话讲清楚 + 一句话点评。30-80 字一条，最多 6 条"],
  "timeline": [
    {"time": "MM-DD HH:mm", "event": "事件描述，包含时间、主体、行动"}
  ],
  "signals": ["值得关注的信号或苗头，每条说明为什么值得关注，不空洞"],
  "risks": ["风险判断：可能恶化的方向、具体路径、为什么"],
  "unknowns": ["信息缺口：会影响判断但目前不清楚的关键问题"],
  "editorReview": "结尾深度复盘 150-300 字。承担'宏观判断'职责。不要重复 overview，要往前推一层：这组新闻怎么改变了之前的认知框架、哪些关联被忽视、读者带走的核心 take。"
}

写作要求：
- keyDevelopments：高关注度最多 ${kdHigh} 条，中关注度最多 ${kdMedium} 条。按"对读者重要"排序，不按时间。
- ${lowHandling === 'drop' ? '低关注度素材直接丢弃，不要出现在任何字段' : lowHandling === 'expand' ? '低关注度也可放入 keyDevelopments（importance: medium）' : '低关注度素材进 briefs 区（短讯），不要塞进 keyDevelopments'}
- 命中读者排除主题的素材：${lowHandling === 'drop' ? '直接丢弃' : '降到 briefs 或不提'}
- briefs 每条要有"【地区】" 前缀，便于扫读
- timeline 按时间正序，事件描述要完整自洽（脱离上下文也能看懂）
- signals/risks/unknowns 各 2-4 条，每条要有"为什么"
- editorTake 是个人观点，可以带情绪、可以辛辣，但要有素材依据

新闻素材：

${itemsText}`
}
```

## 四、output 阶段重写 Markdown 模板

### [src/output.js:59](src/output.js#L59) `buildMarkdown` 整段重写

```js
function buildMarkdown(report, items, config) {
  const date = todayStr()
  const generatedAt = new Date().toISOString()
  const editorial = config.editorial || {}
  const mergeContext = editorial.mergeContextIntoOverview !== false

  // TL;DR 速读
  const tldrEnabled = editorial.tldr?.enabled !== false
  const tldr = (report.tldr || []).map(t => `- ${t}`).join('\n') || '（无）'

  // 关键变化：每条三段
  const keyDev = (report.keyDevelopments || []).map((d, i) => {
    const imp = d.importance === 'high' ? '🔴 高关注' : '🟡 中等'
    // 兼容旧字段：如果 LLM 仍返回 detail 而没返回 what/why/editorTake，降级显示
    if (d.what || d.why || d.editorTake) {
      return `### ${i + 1}. ${d.title} ${imp}

**发生了什么**：${d.what || '（无）'}

**为什么重要**：${d.why || '（无）'}

**编辑怎么看**：${d.editorTake || '（无）'}`
    }
    return `### ${i + 1}. ${d.title} ${imp}

${d.detail || '（无）'}`
  }).join('\n\n---\n\n') || '（无）'

  // 短讯
  const briefs = (report.briefs || []).map(b => `- ${b}`).join('\n')

  const timeline = (report.timeline || []).map(t => `- ${t.time} ${t.event}`).join('\n') || '（无）'
  const signals = (report.signals || []).map(s => `- ${s}`).join('\n') || '（无）'
  const risks = (report.risks || []).map(r => `- ${r}`).join('\n') || '（无）'
  const unknowns = (report.unknowns || []).map(u => `- ${u}`).join('\n') || '（无）'

  const sources = items.map(item => `- [${item.source}: ${item.title}](${item.url})`).join('\n')

  // 拼装：可选段落用条件渲染
  const sections = []
  sections.push(`# ${config.title}`)
  if (tldrEnabled && report.tldr?.length) {
    sections.push(`## 30 秒速读\n\n${tldr}`)
  }
  sections.push(`## 本期概览\n\n${report.overview || '（无）'}`)
  sections.push(`## 关键变化\n\n${keyDev}`)
  if (briefs) {
    sections.push(`## 今日短讯\n\n${briefs}`)
  }
  if (!mergeContext && report.context) {
    sections.push(`## 整体背景\n\n${report.context}`)
  }
  sections.push(`## 时间线\n\n${timeline}`)
  sections.push(`## 值得关注的信号\n\n${signals}`)
  sections.push(`## 风险判断\n\n${risks}`)
  sections.push(`## 信息缺口\n\n${unknowns}`)
  sections.push(`## 主编复盘\n\n${report.editorReview || '（无）'}`)
  sections.push(`## 来源\n\n${sources}`)

  // frontmatter 增加 itemsUsed 字段（粗略估算）
  const itemsUsed = (report.keyDevelopments?.length || 0) + (report.briefs?.length || 0)
  const itemsDropped = Math.max(0, items.length - itemsUsed)

  return `---
topic: ${config.id}
title: ${config.title}
date: ${date}
generatedAt: ${generatedAt}
sourceCount: ${items.length}
itemsUsed: ${itemsUsed}
itemsDropped: ${itemsDropped}
---

${sections.join('\n\n')}
`
}
```

> 关键的向后兼容点：`if (d.what || d.why || d.editorTake)` 判断让旧 prompt 返回的 `detail` 字段仍然能渲染，方便灰度。

## 五、LLM 参数

[src/llm.js:10](src/llm.js#L10) 把 `temperature` 从 0.3 调到 0.6。私人内参风格需要一些锐度，0.3 太冷淡。

```js
function createModel() {
  return new ChatOpenAI({
    apiKey: process.env.LLM_API_KEY,
    model: process.env.LLM_MODEL || 'gpt-4o-mini',
    temperature: Number(process.env.LLM_TEMPERATURE) || 0.6,  // ← 改这里
    configuration: { baseURL: process.env.LLM_BASE_URL },
  })
}
```

> 用 env 而非 yaml：温度是模型层参数，全局统一更合理。如果用户希望按主题调，后续可在 `editorial.llm.temperature` 加配置。

**模型选择**：gpt-4o-mini 应能撑住新 schema（字段更多但每段更短），先保持。如果实测发现字段遵循度差，再升 gpt-4o（成本约 6×）。**这次先不动**。

**max_tokens**：先不加上限。新 schema 输出反而更短（150+200×6+80×6 ≈ 1900 字），比旧版（150+200×5+200+300=1650+ 但实际溢出严重）相当。

## 九、风险与权衡

| 风险 | 评估 | 应对 |
|------|------|------|
| **gpt-4o-mini 字段遵循度不够**，新 schema 字段更多 | 中等 | 先试跑 2 次；若 editorTake/briefs 经常缺失，升 gpt-4o 或加 OpenAI structured output |
| **temperature 0.6 偶尔出现编造判断** | 中 | system prompt 反复强调"只基于素材"；审计日志可对照素材 review |
| **excludeKeywords 误伤**（比如 NHS 关键词把和地缘有关的 NHS 资助案误删） | 低 | 关键词写得保守一些；如发现误伤再调 |
| **lowAttentionHandling=brief 时 LLM 拿不准什么算"低"** | 中 | prompt 已给出明确规则（命中 excludeTopics 必入 briefs）；首跑后观察分布微调 |
| **editorTake 语气过冲、出现敏感判断** | 低-中 | 私人简报场景可接受；如果担心，把 tone 改成"克制但有判断" |
| **TL;DR 和 overview 重复** | 中 | prompt 强调 TL;DR 是"事实速览"、overview 是"主线叙述"，分工明确；如重复就把 overview 改成更聚焦"为什么这组事重要" |

**可分阶段上线**：
- **Stage 1**（小改动，立即可见）：fetch 加 excludeKeywords + LLM temperature 调高。
- **Stage 2**（中等改动）：重写 prompt + 新 JSON schema。
- **Stage 3**（最大改动）：重写 Markdown 模板，加 TL;DR/短讯/三段结构。

如果担心一次性全改风险大，可以按阶段提交三个 commit。

## 主要决策点（一句话总结）

1. **prompt 风格**：从"资深主编"改成"私人内参编辑"，温度 0.3 → 0.6

4. **筛选双层**：yaml `editorial.excludeKeywords`（fetch 硬过滤） + `editorial.excludeTopics`（LLM 软筛选）
5. **配置化**：所有上述行为通过 yaml `editorial:` 段控制，可独立调风格；缺失走默认，向后兼容
