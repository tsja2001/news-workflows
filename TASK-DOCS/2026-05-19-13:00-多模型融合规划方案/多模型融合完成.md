## 新功能是什么

### 核心架构：研究助理 + 主编

```
RSS/HTML/API 抓取 → 过滤管线 → DeepSeek 预处理 → Claude 最终成稿 → output.js
```

### 如何启用

在 topic yaml 中配置：

```yaml
llmPipeline:
  mode: hybrid        # single | hybrid
  preprocess:
    maxInputItems: 80
    itemExcerptChars: 1000
    maxClusters: 12
  writer:
    maxSourceItems: 35
    onFailure: fail   # fail | fallbackToPreprocessModel
  validation:
    enabled: false
```

在 `.env` 中配置：

```env
# 预处理模型（缺失回退 LLM_*）
LLM_PREPROCESS_MODEL=deepseek-v4-pro
LLM_PREPROCESS_TEMPERATURE=0.2

# 写作模型（缺失回退 LLM_*）
LLM_WRITER_MODEL=claude-sonnet-4-6
LLM_WRITER_TEMPERATURE=0.7
```

### 向后兼容

- 未配置 `llmPipeline` 或 `mode: single` 时，行为与之前完全一致
- 未配置 `LLM_PREPROCESS_*` 或 `LLM_WRITER_*` 时，自动回退到 `LLM_*`
- 审计日志 `summary.llm` 保留旧字段（model/inputTokens/outputTokens），同时新增 `stages[]`
- audit CLI 的 `show` 命令自动检测是单阶段还是多阶段

### 失败回退

- 预处理失败 → 自动回退到单模型模式
- 最终成稿失败 → 默认 `fail`；也可配置 `fallbackToPreprocessModel` 回退

## 关键决策与权衡

1. **为什么用动态 import** — `summarize.js` 在 hybrid 模式下用 `await import()` 懒加载子模块，避免 `single` 模式下不必要的模块加载。这是 ES Module 原生特性，不影响测试。

2. **DeepSeek prompt 设计** — 强调"不是主编，只是研究助理"，每个判断绑定 itemId，输出克制简短。避免 DeepSeek 的二手判断影响 Claude 的独立选题。

3. **Claude prompt 设计** — 同时收到 DeepSeek 中间稿和精简原始素材（标题+摘要+URL），确保能重新判断而不仅依赖 DeepSeek。

4. **auditor 向后兼容** — `summary.llm` 同时有 `stages[]`（多阶段详情）和 `model/inputTokens/outputTokens`（汇总），旧 audit CLI 命令不会失效。

5. **校验暂不做 LLM 深度检查** — Phase 1 只做代码级 schema 修复和 excludeTopics 检查。LLM 深度校验（事实越界、编造检测）留到二期。
